Documentation

1. Title:

A Movies Web Application

2. What the API does:

It will provide users with access to information about different movies, directors, and genres

3. List of URL endpoints and request methods

Business Logic URL HTTP Method Request body data format Response body data format
Return a list of all movies /movies GET None A JSON object holding data about all the movies
Return data about a single movie /movies/:title GET None A JSON object holding data about a single movie. Example:
Return data about a genre /movies/genre/:genreName GET None A JSON object holding data about a genre
Return data about a director /movies/directors/:directorName GET None A JSON object holding data about a director
Allow new users to register /users POST A JSON object holding data about the user to add, structured like: 
                    {
                    "name": "Mary",
                    "favoriteMovies":[]
                }
A JSON object holding data about the user that was added, structured like: 
                        {
                        "name": "Mary",
                        "favoriteMovies":[],
                        "id": "388bf9b0-c2b6-4f21-ad01-ebd74ca4ccba"
                    }
Allow users to update their user info /users/:id PUT None A JSON object with the updated info
Allow users to add a movie to their list of favorites /users/:id/:movieTitle PUT None Send user a message that the new movie has been added to their list
Allow users to remove a movie to their list of favorites /users/:id/:movieTitle DELETE None Send user a text message informing them that the movie has been deleted from their list
Allow existing users to de-register /users/:id DELETE None Send a text message indicating that the user has been deleted
